# Master Delivery Plan

## Goal
Deliver a production-ready onboarding and financing platform for partners and internal teams, ending with controlled deployment on GCP and Vercel.

## Workstreams
1. Platform foundation (auth, schema, API standards, CI).
2. Onboarding workflow engine (Stage 1-4 orchestration).
3. Partner and admin experiences by role.
4. Integrations and background jobs (CRB, WhatsApp, device SDK, retries).
5. Security/compliance/audit controls.
6. Testing, observability, release hardening.

## Execution order
- Phase 1: Foundation and environment readiness.
- Phase 2: Onboarding domain implementation.
- Phase 3: Partner/admin product surfaces.
- Phase 4: Reliability, observability, security hardening.
- Phase 5: Deployment, cutover, and post-launch stabilization.

## Success conditions
- End-to-end onboarding reaches "Complete" with auditable evidence.
- Role-based permissions enforced for Owner, Developers, Analyst, Marketers, and Partners.
- GCP/Vercel deployment pipeline is automated and rollback-capable.
