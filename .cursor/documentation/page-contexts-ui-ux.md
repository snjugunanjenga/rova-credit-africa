# UI/UX Context Map by Page

This map defines user intent, UX priorities, and content tone for each page in the current app.

## `/` Home
- Primary intent: convert first-time visitors into marketplace browsing and financing applications.
- UX priorities: fast understanding, trust cues, clear CTA hierarchy, mobile readability.
- Tone: confident, local, practical, transparent financing.

## `/marketplace`
- Primary intent: help users discover a suitable phone and understand affordability quickly.
- UX priorities: easy filtering, consistent cards, visible repayment context, fast scanning.
- Tone: helpful, clear, comparison-friendly.

## `/marketplace/$id`
- Primary intent: remove purchase hesitation for a specific device.
- UX priorities: financing clarity, trust proof, concise specs, direct action buttons.
- Tone: precise, confident, no hidden surprises.

## `/partners`
- Primary intent: convert merchants/sellers into onboarding submissions.
- UX priorities: clear value proposition, transparent economics, low-friction multi-step form.
- Tone: business-professional, mutually beneficial.

## `/about`
- Primary intent: establish trust and legitimacy.
- UX priorities: narrative clarity, leadership credibility, partner/network confidence.
- Tone: mission-driven, credible, East Africa grounded.

## `/contact`
- Primary intent: get user support/intake quickly with confidence.
- UX priorities: channel clarity (WhatsApp/email/location), short validated forms.
- Tone: responsive, supportive, human.

## `/legal/*`
- Primary intent: satisfy compliance and disclosure transparency.
- UX priorities: readability, navigation clarity, non-distracting layout.
- Tone: formal, explicit, policy-oriented.

## `/admin/*`
- Primary intent: operational efficiency for internal roles.
- UX priorities: density with clarity, role actions, status visibility, low visual noise.
- Tone: functional, decisive, accountable.
