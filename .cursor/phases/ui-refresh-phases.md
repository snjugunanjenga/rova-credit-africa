# UI Refresh Phases

## Phase U0: Brand and UX Baseline
Objectives:
- Finalize color tokens, logo usage rules, spacing scale, and typography scale.

Deliverables:
- Updated theme tokens in `src/styles.css`.
- Brand asset intake checklist complete.
- UI reference documentation complete.

Exit criteria:
- Token table approved.
- Header/footer logo integration plan approved.

## Phase U1: Shared Layout and Navigation
Objectives:
- Update site shell, header, footer, global buttons, chips, and card primitives.

Deliverables:
- Header/footer with Rova logo.
- Unified nav behavior for desktop + mobile.
- Updated CTA/button hierarchy classes.

Exit criteria:
- Shared layout consistent on all public routes.
- Mobile nav and sticky elements pass interaction checks.

## Phase U2: Public Conversion Pages
Objectives:
- Implement refreshed home, marketplace, product detail, partners, about, and contact pages.

Deliverables:
- Page-level UI parity with approved direction.
- Updated empty/loading/error states.
- Consistent spacing and hierarchy patterns.

Exit criteria:
- All public page acceptance criteria pass.
- Conversion CTAs visually and functionally verified.

## Phase U3: Legal/Admin Alignment
Objectives:
- Apply lighter brand alignment to legal/admin pages without reducing readability/operability.

Deliverables:
- Legal readability updates.
- Admin brand polish updates.

Exit criteria:
- No admin workflow friction introduced.
- Legal readability/accessibility preserved.

## Phase U4: QA, Accessibility, and Launch Readiness
Objectives:
- Validate UX quality, accessibility, and regression safety.

Deliverables:
- Playwright coverage for key journeys.
- Accessibility and contrast audit report.
- UI release checklist completion.

Exit criteria:
- E2E critical tests green.
- Accessibility issues triaged/resolved for launch scope.
