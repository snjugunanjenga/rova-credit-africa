# Implementation Plan

## Stream A: Foundation
- Set up auth, permissions, and tenancy model.
- Establish core schema and migration pipelines.
- Implement baseline API skeleton and health endpoints.

## Stream B: Onboarding Domain
- Build stage state machine and transition APIs.
- Integrate CRB checks, device enrollment, and agreement dispatch.
- Add media upload and verification workflow.

## Stream C: Role Experiences
- Partner intake and status tracking UI.
- Admin work queues and override controls by role.
- Analytics views for analysts and marketers.

## Stream D: Quality and Operations
- Add unit/integration/E2E tests.
- Instrument logs, metrics, traces, and alerts.
- Finalize runbooks and incident playcards.
