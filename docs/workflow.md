# Application Workflow

## Primary Workflow: Client Financing Onboarding
1. Partner creates application for a client.
2. Eligibility service performs CRB risk assessment.
3. Approved applications progress to device/downpayment collection.
4. KYC evidence and setup validations are completed.
5. Agreement is signed and activation is confirmed.
6. Application moves to complete state with audit trail.

## Operational Workflow: Admin Review
- Queue-driven review for exceptions at each stage.
- Escalation paths depend on role permissions.
- Rejected or amended records generate partner feedback tasks.

## Communication Workflow
- WhatsApp templates notify partner/client at each major transition.
- Failures trigger retries and fallback admin alerts.
